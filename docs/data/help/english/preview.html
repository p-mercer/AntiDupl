Preview Panel

Review panel at the table of vertical pairs view mode of results is located in the left side of the main window:

or at the top of the main window when table of horizontal pairs view mode of results:

Switching the view mode of results is carried out using the item View - View mode of results of Main Menu. The functionality of the preview pane does not depend on the view mode of results and is designed to display the current result of Table of Results. With it you can examine in detail the duplicate or defective images, as well as make the basic operations on them.

Showing of pair of duplicates

A pair of duplicate images is displayed on the panel in a column or row (depending form view mode of results). The relative position of images corresponds to their position in the list of results. The control buttons are placed near them. Next is a list of these buttons and their names:

  • or - Delete both pictures.

  • or - Replace second picture by the first picture.

  • or - Delete first picture.

  • - Go to the previous result.

  • - Go to the next result.

  • or - Delete second picture.

  • or - Replace first picture by the second picture.

  • - Mark current result as a mistake.

The meanings of most actions undertaken by the buttons are well understood from their names. We describe only the last: this button allows you to mark the current result as false that would exclude his appearance after repeated searches. Additionally, we note that all the above actions (except for moving between the results) may be undone.

howing of defect picture

Defective image is displayed at the top panel or at the left panel (depending form view mode of results). It might happen that the program can not open it and the top panel will be a blank image. There are control buttons near:

  • or - Delete picture.

  • - Go to the previous result.

  • - Go to the next result.

  • - Mark current result as a mistake.

Additional features

  • Under each picture is the panel with some information about it. It includes file size of image on the disk, the size (width and height), blockiness of the image, bluring of the image, its type, EXIF information and the path to the image. Note that if a duplicate image pairs have a different type, different EXIF information then this information is highlighted in red. Also highlighted in red image information having a smaller dimensions or smaller file size. EXIF information is showing in tool tip to a text of 'EXIF' on the following fields: description, equipment make, equipment model, software, date/time, artist, comment. Double-clicking on the name of the picture, you can rename or move a given image.

  • When you double-click the mouse on the picture, it will be opened with using the external picture viewer.

  • In some cases, when the program can uniquely identify a better picture of the pair, it will create a recommendation for the user, and the corresponding button will be marked in pink.

  • All buttons on this panel are duplicated by the Hot Keys. Current hot keys appear in the description of the buttons in tooltips, if you move the mouse cursor to any of the buttons on the panel.

  • If you right click on the picture, it opens a context menu with the following items:

    • Copy the path to this picture.

    • Copy a picture name.

    • Open the picture with using an external picture viewer.

    • View the directory, which contains the current image.

    • Add image to the ignored.

    • Add directory of image to the ignored paths. The same can be made and on a tab ignore, windows of a choice of paths.

    • Rename or move current image.

    • Rename image like neighbour image. If the file with such name already exists, renames with a similar name. (In a name digits since the end if are that are looked for the digit increases by unit if aren't, or the file with such digit already is, the character' _2 is added').

    • Move image into the directory of neighboring image. Renaming it if the file with a name moved already is in the directory. This operation is enabled if these images are placed in different directories.

    • Move image into the directory of neighboring image and rename image like neighbour image. This operation is enabled if these images are placed in different directories.

    • Move group of image in neighbour directory. Moves all files from the current group of duplicates to directory of neighboring image. If the file with a name of the transferable file already is in the directory, the transferable file is renamed into the file with a similar name. This operation is enabled if these images are placed in different directories.

    • Rename group of image as neighbour image. Rename names of all files from the current group of duplicates on names similar at neighbour image name.

    • Note that the move operation or rename the images are undone.
